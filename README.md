# fridge-chef 🧊🍳

Send a photo of your fridge to a Telegram bot and get back 3 recipes based on
what's inside, generated by Gemini.

## How it works

- Runs as a single Docker container using Telegram **long polling** — no public
  IP, domain, or open ports needed.
- Photos are sent to **gemini-2.5-flash-lite** (the cheapest multimodal Gemini
  model). On failure it retries twice with backoff, then falls back to
  **gemini-2.5-flash** once. If both fail you get an error message in Telegram.
- Albums (multiple photos sent together) are combined into a **single** request
  producing one set of 3 recipes.
- Recipes are in **Turkish** by default; add an English caption to the photo to
  get them in English. Captions are also treated as constraints
  (e.g. "vejetaryen", "30 dakikada", "vegetarian, quick").
- Recipes use what's visible in the fridge plus common pantry staples
  (marked "(kiler)").
- Only Telegram user IDs on the allowlist can use the bot; everyone else is
  silently ignored.
- **No repeats**: suggested recipe names are stored in a SQLite database
  (`data/history.db`, persisted via a volume mount) and dishes from the last
  5 days are excluded from new suggestions via the prompt. History is shared
  by all allowed users (one fridge, one household) and is best-effort — a
  database problem never blocks recipe generation.
- `/recent` lists the dishes suggested within the window; `/forget` clears
  the history.

## Setup

1. **Create the bot**: message [@BotFather](https://t.me/BotFather) on
   Telegram, send `/newbot`, follow the prompts, and copy the token.
2. **Find your Telegram user ID**: message
   [@userinfobot](https://t.me/userinfobot) — it replies with your numeric ID.
3. **Get a Gemini API key**: https://aistudio.google.com/apikey
4. **Configure**:

   ```bash
   cp .env.example .env
   # edit .env and fill in the three values
   ```

5. **Run**:

   ```bash
   docker compose up -d --build
   ```

6. Open your bot in Telegram, send `/start`, then send a fridge photo.

## Operations

```bash
docker compose logs -f     # watch logs
docker compose restart     # restart
docker compose down        # stop
docker compose up -d --build   # rebuild after changing code or .env models
```

## Configuration reference

| Variable | Required | Default | Description |
|---|---|---|---|
| `TELEGRAM_BOT_TOKEN` | yes | — | Bot token from @BotFather |
| `GEMINI_API_KEY` | yes | — | Google AI Studio API key |
| `ALLOWED_USER_IDS` | yes | — | Comma-separated numeric Telegram user IDs |
| `MAIN_MODEL` | no | `gemini-2.5-flash-lite` | Model tried first (3 attempts with backoff) |
| `FALLBACK_MODEL` | no | `gemini-2.5-flash` | Model tried once if the main model fails |
| `HISTORY_DAYS` | no | `5` | Days a suggested recipe stays off the menu |
| `HISTORY_DB` | no | `data/history.db` | SQLite file for suggestion history |
